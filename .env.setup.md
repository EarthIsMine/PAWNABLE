# 환경 변수 설정 가이드

## 📋 설정 방법

1. `.env.example` 파일을 복사하여 `.env` 파일 생성:
   ```bash
   cp .env.example .env
   ```

2. `.env` 파일을 열고 실제 값으로 수정:
   ```bash
   nano .env
   # 또는
   vim .env
   ```

## 🔑 필수 환경 변수

### Server Configuration
- `PORT`: 서버 포트 (기본값: 8085)
- `NODE_ENV`: 실행 환경 (development, production)

### Database Configuration
- `DB_HOST`: PostgreSQL 호스트 (기본값: localhost)
- `DB_PORT`: PostgreSQL 포트 (기본값: 5432)
- `DB_USERNAME`: PostgreSQL 사용자명
- `DB_PASSWORD`: ⚠️ **PostgreSQL 비밀번호 (실제 값으로 변경)**
- `DB_DATABASE`: 데이터베이스 이름 (기본값: pawnable_db)

### JWT Configuration
- `JWT_SECRET`: ⚠️ **JWT 시크릿 키 (실제 값으로 변경)**
  - 개발: 임의의 강력한 문자열
  - 프로덕션: 최소 32자 이상의 랜덤 문자열
- `JWT_EXPIRES_IN`: JWT 만료 시간 (기본값: 7d)

### CORS Configuration
- `CORS_ORIGIN`: 허용할 프론트엔드 도메인
  - 개발: http://localhost:3001
  - 프로덕션: https://your-domain.com

## 🔒 보안 주의사항

1. **절대 .env 파일을 Git에 커밋하지 마세요**
   - 이미 `.gitignore`에 포함되어 있습니다

2. **프로덕션 환경에서는 강력한 비밀번호 사용**
   ```bash
   # 랜덤 시크릿 생성 예시
   openssl rand -base64 32
   ```

3. **환경별로 다른 값 사용**
   - 개발/스테이징/프로덕션 환경마다 다른 시크릿 사용

## ✅ 설정 확인

서버 시작 후 다음 엔드포인트로 확인:
```bash
curl http://localhost:8085/health
```

정상 응답:
```json
{
  "success": true,
  "message": "Server is healthy",
  "timestamp": "2025-12-20T14:48:43.365Z"
}
```
